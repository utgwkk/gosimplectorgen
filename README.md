# gosimplectorgen

A tiny Go code generator that emits `NewXxx` constructors for struct types in a file.

Given a Go source file containing struct type declarations, `gosimplectorgen` generates a companion `*_gen.go` file with one constructor function per struct. Each constructor takes all named fields as positional arguments and returns a pointer to a fully initialized struct literal.

## Installation

### go install

```sh
go install github.com/utgwkk/gosimplectorgen@latest
```

### Go 1.24+ module tool (recommended)

Add the tool to your module's `go.mod`:

```sh
go get -tool github.com/utgwkk/gosimplectorgen@latest
```

Then invoke it via:

```sh
go tool gosimplectorgen <input.go>
```

## Usage

```
gosimplectorgen [-out <output.go>] [-targetTypes <Type1,Type2>] <input.go>
```

| Flag | Default | Description |
|------|---------|-------------|
| `-out` | `<input>_gen.go` | Output file path |
| `-targetTypes` | _(all structs)_ | Comma-separated list of struct type names to generate constructors for |

The input must be a single `.go` file. The generated file is placed in the same package.

### `go:generate` integration

Place a `//go:generate` directive in the file you want to process:

```go
//go:generate go tool gosimplectorgen $GOFILE
```

Then run `go generate ./...` to (re)generate.

## Example

**Input** (`def.go`):

```go
package example

import (
	"github.com/utgwkk/gosimplectorgen/example/bar"
	"github.com/utgwkk/gosimplectorgen/example/baz"
	"github.com/utgwkk/gosimplectorgen/example/foo"
)

//go:generate go tool gosimplectorgen $GOFILE

type privateStruct struct {
	foo foo.Foo
	bar bar.Bar
	baz baz.Baz
}
```

**Generated output** (`def_gen.go`):

```go
// Code generated by github.com/utgwkk/gosimplectorgen. DO NOT EDIT.

package example

import (
	"github.com/utgwkk/gosimplectorgen/example/bar"
	"github.com/utgwkk/gosimplectorgen/example/baz"
	"github.com/utgwkk/gosimplectorgen/example/foo"
)

func NewPrivateStruct(
	foo foo.Foo,
	bar bar.Bar,
	baz baz.Baz,
) *privateStruct {
	return &privateStruct{
		foo: foo,
		bar: bar,
		baz: baz,
	}
}
```

A live copy of this example lives in the [`example/`](./example) directory.

## Behavior & limitations

- Fields become positional parameters in declaration order. Multi-name field declarations (`a, b int`) are expanded into separate parameters.
- Embedded (anonymous) fields are skipped.
- Field types are preserved from the original AST, so generics, pointers, slices, maps, channels, and function types all work without special handling.
- All `import` declarations from the source file are copied verbatim into the generated file, even if no constructor references them. In pathological cases this may produce an unused-import compile error.
- Constructor names are always exported (`NewXxx`), even when the struct itself is unexported.
